## ⏱️ 超时设置

### 根据任务类型动态设置

| 任务类型 | 超时时间 | 示例 |
|---------|---------|------|
| **简单查询** | 60 秒 | "查一下股价" |
| **投资分析** | 300 秒 | "分析腾讯控股" |
| **软件开发** | 600 秒 | "开发一个 API" |
| **内容创作** | 300 秒 | "写篇文章" |
| **复杂分析** | 600 秒 | "深度研究报告" |

### 配置规则

```python
def get_timeout(task_type):
    timeouts = {
        'simple_query': 60,
        'investment_analysis': 300,
        'software_dev': 600,
        'content_creation': 300,
        'complex_analysis': 600
    }
    return timeouts.get(task_type, 300)
```

## 📋 任务分解策略

### 大任务分解为小任务

**示例：投资分析任务**

```
原任务：分析腾讯控股投资价值（需要 5-10 分钟）

分解为：
1. 获取公司基本信息（60 秒）
2. 获取财务数据（60 秒）
3. 分析商业模式（60 秒）
4. 评估护城河（60 秒）
5. 估值分析（60 秒）
6. 汇总投资建议（60 秒）
```

## 🔄 重试机制

### 失败重试策略

```
第一次失败 → 等待 10 秒 → 重试
第二次失败 → 等待 30 秒 → 重试
第三次失败 → 报告错误 → 人工介入
```

### 重试条件

- ✅ 网络超时 → 重试
- ✅ 临时错误 → 重试
- ❌ 配置错误 → 不重试
- ❌ 权限问题 → 不重试
