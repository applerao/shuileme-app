## 02 AI 的"手"，终于长出来了

过去两年，大模型火了。

但你有没有发现一个问题：**AI 只会动嘴，不会动手**。

你能跟它聊哲学、聊代码、聊人生，但让它帮你：

- 查一下明天的天气
- 把这份 Excel 整理一下
- 给团队发个通知
- 监控某个网页的价格变化

**抱歉，它做不到。**

因为它没有"手"。

OpenClaw 做的事，就是**给 AI 装上一双手**。

```
# 以前
用户：明天北京天气怎么样？

# 现在
用户：明天北京天气怎么样？
AI:  明天北京晴转多云，15-25°C，适合出门 🌤️
```

这不是魔法，这是**工具调用**。

OpenClaw 内置了一套完整的工具系统：

| 工具 | 能力 |

| 文件工具 | 读/写/编辑/管理任意文件 |

| 终端 | 执行 Shell 命令、跑脚本 |
| 消息 | 钉钉/Telegram/Discord/微信 |
| 子代理 | 任务编排、多 Agent 协作 |

**AI 不再是聊天机器人，它是你的数字助理。**

## 06 安全吗？

这是被问得最多的问题。

**"给 AI 这么多权限，不会把我的文件删了吧？"**

OpenClaw 的安全机制：

1. **敏感操作需要确认** - 删除、外部发送等操作会弹出确认
2. **工具调用有策略** - 可以配置哪些工具能用，哪些不能用
3. **子代理隔离** - 每个子代理运行在独立会话中
4. **日志可追溯** - 所有操作都有记录

**当然，最基本的原则还是：不要给 AI 太高权限。**

就像你不会把 root 密码告诉别人一样，也不要让 AI 随意访问敏感目录。

## 08 怎么开始？

### 第一步：安装

```bash
npm install -g openclaw
```

需要 Node.js 18+，Linux/macOS/WSL 环境。

### 第二步：配置

根据你想用的渠道，配置对应的凭证：

- 钉钉：创建机器人，获取 Webhook
- Telegram：找 BotFather 创建 Bot
- Discord：创建应用，获取 Token

### 第三步：启动

```bash
openclaw gateway start
```

### 第四步：开聊

在对应的渠道里，给你的 AI 助理发消息：

```
"帮我查一下明天北京的天气"
"把这个文件夹里的文件按日期整理一下"
"每天上午 9 点提醒我看日报"
```

**就这么简单。**
