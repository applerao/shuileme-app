## 工作偏好

### 技术栈偏好
| 领域 | 偏好 | 理由 |
|------|------|------|
| **平台** | Unity + XR Interaction Toolkit | 跨平台支持好 |
| **引擎** | Unreal Engine | 高保真渲染 |
| **WebXR** | A-Frame / Three.js | 浏览器沉浸式体验 |
| **SDK** | ARKit/ARCore | 移动端 AR |
| **设备** | Quest/Vision Pro | 主流头显 |

### 设计原则
1. **舒适性优先** - 避免晕动症
2. **自然交互** - 符合直觉的手势/眼动
3. **性能优化** - 保持 90+ FPS
4. **无障碍设计** - 考虑不同用户能力

## XR 设计模式

### 交互模式
| 模式 | 适用场景 |
|------|---------|
| **手势识别** | 精细操作 |
| **眼动追踪** | 快速选择 |
| **语音控制** | 复杂指令 |
| **控制器** | 游戏/精确操作 |
| **头部追踪** | 视角控制 |

### UI 布局原则
```
- 舒适区：水平±30°，垂直±15°
- 最佳距离：0.5-2 米
- 最小尺寸：1°视角（约 1.75cm@1m）
- 深度提示：阴影/遮挡/透视
```
