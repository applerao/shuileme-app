## 工作偏好

### 排查方法论
1. **由点及面** - 从单个 bug 扩展到同类问题
2. **分层扫描** - 按文件类型/功能模块系统排查
3. **优先级排序** - 按影响范围和严重程度分级
4. **根因分析** - 不止修复表面，找到根本原因

### 报告标准
- ✅ 问题清单（含文件路径和行号）
- ✅ 严重程度分级（P0-P3）
- ✅ 修复建议
- ✅ 影响范围评估

## 历史经验

### 2026-03-12 - 被裁了吗 App SQLite 兼容性全面排查

**任务：** 从单个编译错误扩展到全面排查 PostgreSQL 兼容性问题

**排查过程：**

#### 第 1 轮：编译错误
```
错误：CreateUserDTO 接口定义不匹配
位置：src/models/User.ts:15
修复：password → password_hash
```

#### 第 2 轮：运行时错误
```
错误：RETURNING * 不支持
位置：src/models/User.ts, src/models/Checkin.ts
影响：所有 INSERT/UPDATE 操作
修复：插入后重新查询
```

#### 第 3 轮：全面扫描
```bash
# 搜索所有 PostgreSQL 特有语法
grep -r "RETURNING" src/
grep -r "::date" src/
grep -r "NOW()" src/
grep -r "ROW_NUMBER()" src/
```

**发现的问题：**
| 文件 | 问题 | 严重度 | 状态 |
|------|------|--------|------|
| `User.ts` | RETURNING * | P0 | ✅ 已修复 |
| `User.ts` | NOW() 函数 | P1 | ✅ 已修复 |
| `Checkin.ts` | RETURNING * | P0 | ✅ 已修复 |
| `Checkin.ts` | ::date 转换 | P1 | ✅ 已修复 |
| `Checkin.ts` | 窗口函数 | P2 | ✅ 已修复 |
| `Checkin.ts` | 参数传递 | P0 | ✅ 已修复 |

**学到的经验：**
1. ✅ 系统排查比单点修复更有效
2. ⚠️ 初期排查范围不够，遗漏了 getStats 的参数问题
3. ✅ 建立问题清单，逐个击破
4. ✅ 修复后立即验证，避免回归

**产出文档：**
- `SQLITE_COMPATIBILITY.md` - 完整兼容性报告
- 问题追踪清单（已集成到文档）
