## 工作偏好

### 技术栈偏好
| 领域 | 偏好 | 理由 |
|------|------|------|
| **容器** | Docker + Docker Compose | 简单够用 |
| **编排** | Kubernetes（生产） | 可扩展性 |
| **CI/CD** | GitHub Actions | 与代码仓库集成 |
| **监控** | Prometheus + Grafana | 开源成熟 |
| **日志** | ELK Stack | 功能完整 |
| **云服务商** | 阿里云 | 本地化支持好 |

### 运维原则
1. **自动化优先** - 能自动化就不手动
2. **可回滚** - 所有部署支持一键回滚
3. **监控先行** - 上线前先配置监控
4. **文档完整** - 运维操作有文档

### 交付物标准
- ✅ 可执行的部署脚本
- ✅ CI/CD配置文件
- ✅ 监控告警规则
- ✅ 运维手册（Runbook）
- ✅ 灾难恢复方案

## 监控告警标准

### 核心指标
| 指标 | 告警阈值 | 响应时间 |
|------|---------|---------|
| CPU 使用率 | > 80% | 15 分钟 |
| 内存使用率 | > 85% | 15 分钟 |
| 磁盘使用率 | > 90% | 1 小时 |
| 接口错误率 | > 5% | 5 分钟 |
| 响应时间 P99 | > 2s | 30 分钟 |

### 告警渠道
1. 钉钉/企业微信（即时）
2. 邮件（非紧急）
3. 电话（P0 级别）
