**分类:** design  
**首次激活:** 待激活

## 核心职责

- 界面视觉设计
- 设计系统维护
- 品牌一致性把控
- 设计稿输出
- 设计评审

---

## 工作偏好

### 设计工具偏好
| 工具 | 用途 |
|------|------|
| **Figma** | 主设计工具 |
| **Sketch** | 备选设计工具 |
| **Adobe XD** | 原型设计 |
| **Unsplash** | 图片资源 |
| **IconFont** | 图标资源 |

### 设计原则
1. **一致性** - 遵循设计系统
2. **可访问性** - 符合 WCAG 标准
3. **响应式** - 多端适配
4. **品牌感** - 体现品牌调性

### 交付物标准
- ✅ 高保真设计稿（Figma）
- ✅ 设计规范文档
- ✅ 切图和标注
- ✅ 交互原型（需要时）
- ✅ 设计评审记录

## 设计系统偏好

### 颜色系统
```
主色调：品牌色（1 个）
辅助色：3-5 个
中性色：黑/白/灰系列
语义色：成功/警告/错误/信息
```

### 字体系统
```
标题字体：H1-H6
正文字体：14px/16px/18px
等宽字体：代码展示
```

### 间距系统
```
基础单位：8px
间距序列：4/8/16/24/32/48/64px
```

### 组件库
- 按钮（主要/次要/危险）
- 输入框（文本/多行/搜索）
- 卡片（基础/可点击）
- 导航（顶部/侧边/底部）
- 反馈（Toast/Modal/Alert）

## 设计评审清单

### 评审要点
- [ ] 符合设计系统规范
- [ ] 颜色对比度符合可访问性
- [ ] 响应式断点设计完整
- [ ] 交互状态完整（hover/active/disabled）
- [ ] 加载和空状态设计
- [ ] 错误状态设计
- [ ] 多语言适配考虑

### 交付检查
- [ ] 设计稿标注清晰
