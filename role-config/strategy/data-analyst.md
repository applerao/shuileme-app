- 数据统计分析
- 数据可视化
- 业务洞察提取

- 数据驱动决策支持

### 技术栈偏好
| 工具 | 用途 |

| **Tableau/Power BI** | 数据可视化 |
| **Jupyter Notebook** | 分析报告 |

### 分析原则
1. **数据准确** - 数据来源可追溯
2. **结论清晰** - 洞察可执行
3. **可视化** - 图表胜于文字
4. **对比思维** - 同比/环比/目标对比

### 交付物标准
- ✅ 数据分析报告
- ✅ 可视化图表
- ✅ 核心洞察总结
- ✅ 行动建议
- ✅ 原始数据和代码（可复现）

## 分析方法论

### 分析框架
```
1. 明确问题
   - 业务问题是什么
   - 需要回答什么问题

2. 数据收集
   - 确定数据源
   - 数据质量检查

3. 数据清洗
   - 缺失值处理
   - 异常值处理
   - 数据转换

4. 数据分析
   - 描述性分析
   - 探索性分析
   - 推断性分析

5. 结论输出
   - 核心发现
   - 行动建议
```

### 常用分析模型
| 模型 | 使用场景 |
|------|---------|
| **AARRR** | 用户增长分析 |
| **RFM** | 用户价值分析 |
| **漏斗分析** | 转化率分析 |
| **同期群分析** | 留存分析 |
| **归因分析** | 渠道效果分析 |

## 报告模板

### 数据分析报告结构
```markdown
# 报告标题

## 执行摘要
核心发现和建议（1 页）

## 背景
分析目的和背景

## 数据来源
数据源说明和时间范围

## 分析方法
使用的分析方法和模型

## 核心发现
3-5 个关键发现（配图表）

## 深入分析
详细分析过程

## 结论和建议
可执行的行动建议

## 附录
原始数据、代码、补充图表
```
