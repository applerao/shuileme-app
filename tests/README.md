# 睡了么 App - 测试套件

本目录包含"睡了么"App 的完整测试资源。

## 目录结构

```
tests/
├── test-cases/              # 测试用例文档
│   └── test-cases.md        # 详细测试用例说明
├── unit-tests/              # 单元测试代码
│   ├── app.test.js          # 主测试文件
│   └── __mocks__/           # Mock 数据（待创建）
├── reports/                 # 测试报告
│   └── test-report-template.md  # 测试报告模板
├── performance/             # 性能测试脚本（待创建）
│   └── api-benchmark.js     # API 性能测试
└── README.md                # 本文件
```

## 快速开始

### 安装依赖

```bash
npm install
```

### 运行测试

```bash
# 运行所有测试
npm test

# 运行特定测试文件
npm test -- app.test.js

# 运行测试并生成覆盖率报告
npm test -- --coverage

# 监听模式运行测试
npm test -- --watch
```

### 性能测试

```bash
# 运行 API 性能基准测试
node tests/performance/api-benchmark.js
```

## 单元测试结构

```javascript
describe('模块名称', () => {
  describe('功能子模块', () => {
    test('TC-XXX-XXX: 测试用例描述', () => {
      // 测试代码
    });
  });
});
```

### 测试用例命名规范

- 格式：`TC-[模块]-[序号]`
- 示例：`TC-REG-001` (注册模块第 1 个用例)

### 模块代码对照

## 测试报告

测试完成后，使用 `reports/test-report-template.md` 模板生成测试报告。

### 报告填写步骤

1. 复制模板到新文件：`test-report-YYYY-MM-DD.md`
2. 填写测试周期和版本信息
3. 统计测试用例执行结果
4. 记录缺陷信息
5. 填写性能测试数据
6. 给出测试结论和建议
7. 相关人员审批签字

## 常见问题

### Q: 测试失败如何处理？

1. 查看错误日志
2. 确认测试环境配置
3. 检查测试数据是否正确
4. 联系开发人员确认是否为缺陷

### Q: 如何添加新测试用例？

1. 在 `test-cases.md` 中添加用例描述
2. 在 `app.test.js` 中实现对应测试代码
3. 更新本文档的用例统计

### Q: 性能测试数据异常？

1. 确认测试服务器负载正常
2. 检查网络环境
3. 多次运行取平均值
4. 对比历史数据趋势
