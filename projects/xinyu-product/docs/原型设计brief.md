## 一、设计目标

### 1.1 核心目标
- 设计一套**温暖、治愈、可信赖**的视觉风格
- 打造**低门槛、流畅**的核心用户体验
- 建立**清晰的信息架构**和**直观的交互流程**

### 1.2 设计风格

| 维度 | 方向 | 参考 |
|------|------|------|
| **色彩** | 温暖柔和、低饱和度 | 莫兰迪色系、暖灰色调 |
| **字体** | 清晰易读、有人情味 | 思源黑体、手写体点缀 |
| **图形** | 圆润、柔和、无攻击性 | 圆角卡片、柔和阴影 |
| **动效** | 流畅、自然、有反馈 | 微动效、过渡动画 |
| **整体感觉** | 治愈、安全、可信赖 | - |

### 1.3 设计原则

1. **隐私可见：** 让用户清晰感知隐私控制权
2. **降低门槛：** 日记写作有引导、有模板
3. **渐进披露：** 信息分层展示，避免 overwhelm
4. **情感化：** 有温度、有共鸣的设计细节

## 三、核心流程设计

### 3.1 注册→测试→首篇日记流程

```
启动页
  ↓
注册/登录页
  ↓
新人引导（3-5 屏）
  ↓
性格测试入口引导
  ↓
性格测试（30-40 题）
  ↓
测试结果页
  ↓
引导写首篇日记
  ↓
选择模板
  ↓
编辑日记
  ↓
设置权限
  ↓
发布成功
  ↓
首页（开始推荐）
```

**关键设计点：**
- 新人引导要简洁、有价值感
- 测试进度要可视化、可暂停
- 日记模板要降低写作门槛
- 权限设置要清晰、易懂

---

### 3.2 匹配→聊天流程

```
首页推荐列表
  ↓
查看用户详情（资料 + 日记片段）
  ↓
点击"心动"（表达喜欢）
  ↓
等待对方回应
  ↓
【匹配成功】通知
  ↓
进入聊天界面
  ↓
破冰话题建议
  ↓
开始聊天
```

**关键设计点：**
- 推荐理由要清晰（为什么匹配）
- 日记片段展示要有吸引力
- 匹配成功要有仪式感
- 聊天开场要有辅助

---

### 3.3 日记发布流程

```
首页/日记 tab → 点击"写日记"
  ↓
选择模板（可选）
  ↓
选择心情
  ↓
编辑内容（文字 + 图片）
  ↓
设置可见权限
  ↓
预览（可选）
  ↓
发布
  ↓
发布成功反馈
```

**关键设计点：**
- 模板选择要直观
- 编辑器要简洁、专注
- 权限设置要醒目
- 发布反馈要有成就感

## 四、交互细节规范

### 4.1 通用交互

| 交互类型 | 规范 |
|----------|------|
| **点击反馈** | 轻微缩放（0.95）+ 颜色变化 |
| **加载状态** | 骨架屏 + 进度提示 |
| **空状态** | 插图 + 引导文案 + 行动按钮 |
| **错误提示** | Toast 提示 + 解决建议 |
| **成功反馈** | Toast + 动效 + 下一步引导 |

### 4.2 隐私交互

| 场景 | 交互设计 |
|------|----------|
| 权限设置 | 三级切换开关 + 说明文案 |
| 公开提示 | 发布前弹窗确认 + 风险提醒 |
| 隐私变更 | 变更通知 + 影响范围说明 |

### 4.3 情感化设计

| 场景 | 设计细节 |
|------|----------|
| 首次写日记 | 庆祝动效 + 鼓励文案 |
| 连续记录 | 成就徽章 + 进度展示 |
| 收到共鸣 | 温暖动效 + 感谢提示 |
| 匹配成功 | 庆祝动效 + 匹配原因 |
| 情绪低落 | 温和色调 + 关怀文案 |
