### 1.2 用户指标

#### 规模指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| 累计用户数 | 历史累计注册用户总数 | COUNT(DISTINCT user_id) | 每日 | - |
| 日新增用户 | 当日新注册用户数 | COUNT(DISTINCT user_id WHERE register_date=today) | 每日 | ≥2,000 |
| 周新增用户 | 当周新注册用户数 | COUNT(DISTINCT user_id WHERE register_date=this_week) | 每周 | ≥10,000 |
| 月新增用户 | 当月新注册用户数 | COUNT(DISTINCT user_id WHERE register_date=this_month) | 每月 | ≥50,000 |

#### 活跃指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| DAU（日活跃用户） | 当日启动 App 的去重用户数 | COUNT(DISTINCT user_id WHERE login_date=today) | 每日 | ≥50,000 |
| WAU（周活跃用户） | 当周启动 App 的去重用户数 | COUNT(DISTINCT user_id WHERE login_date=this_week) | 每周 | ≥200,000 |
| MAU（月活跃用户） | 当月启动 App 的去重用户数 | COUNT(DISTINCT user_id WHERE login_date=this_month) | 每月 | ≥500,000 |
| DAU/MAU | 用户活跃粘性 | DAU / MAU | 每周 | ≥20% |
| 人均使用时长 | 用户平均每日使用时长 | SUM(duration) / DAU | 每日 | ≥15 分钟 |
| 人均启动次数 | 用户平均每日启动次数 | SUM(session_count) / DAU | 每日 | ≥3 次 |

#### 留存指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| 次日留存率 | 新增用户次日仍活跃的比例 | 次日活跃新增数 / 新增用户数 | 每日 | ≥45% |
| 7 日留存率 | 新增用户 7 日后仍活跃的比例 | 第 7 日活跃新增数 / 新增用户数 | 每周 | ≥30% |
| 30 日留存率 | 新增用户 30 日后仍活跃的比例 | 第 30 日活跃新增数 / 新增用户数 | 每月 | ≥20% |
| 90 日留存率 | 新增用户 90 日后仍活跃的比例 | 第 90 日活跃新增数 / 新增用户数 | 每季度 | ≥15% |

#### 流失指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 预警阈值 |
|---------|------|---------|---------|---------|
| 日流失率 | 当日流失用户占比 | 当日流失用户数 / 昨日活跃用户数 | 每日 | >5% |
| 周流失率 | 当周流失用户占比 | 当周流失用户数 / 上周活跃用户数 | 每周 | >15% |
| 月流失率 | 当月流失用户占比 | 当月流失用户数 / 上月活跃用户数 | 每月 | >30% |
| 流失用户数 | 连续 30 天未登录的用户数 | COUNT(user_id WHERE last_login < today-30) | 每周 | - |

### 1.4 商业指标

#### 收入指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| 日营收 | 当日总收入 | SUM(order_amount WHERE pay_date=today) | 每日 | ≥¥50,000 |
| 月营收 | 当月总收入 | SUM(order_amount WHERE pay_date=this_month) | 每月 | ≥¥1,500,000 |
| 付费用户数 | 当日/月付费用户数 | COUNT(DISTINCT user_id WHERE pay_date=today/month) | 每日/月 | - |
| 付费率 | 活跃用户中付费的比例 | 付费用户数 / DAU(or MAU) | 每周 | ≥5% |
| ARPU（每用户平均收入） | 平均每用户贡献收入 | 总收入 / 活跃用户数 | 每月 | ≥¥3 |
| ARPPU（每付费用户平均收入） | 平均每付费用户贡献收入 | 总收入 / 付费用户数 | 每月 | ≥¥60 |

#### 价值指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| LTV（用户生命周期价值） | 用户生命周期内贡献的总收入 | ARPU × 平均生命周期（月） | 每月 | ≥¥180 |
| LTV/CAC | 用户价值与获客成本比 | LTV / CAC | 每月 | ≥3:1 |
| 复购率 | 付费用户中再次付费的比例 | 复购用户数 / 付费用户数 | 每月 | ≥40% |
| 订阅续费率 | VIP 订阅到期后续费的比例 | 续费用户数 / 到期用户数 | 每月 | ≥60% |
| 退款率 | 订单中申请退款的比例 | 退款订单数 / 总订单数 | 每周 | ≤5% |

#### 成本指标

| 指标名称 | 定义 | 计算公式 | 监控频率 | 目标值 |
|---------|------|---------|---------|--------|
| CAC（用户获客成本） | 平均获取一个付费用户的成本 | 营销总费用 / 新增付费用户数 | 每月 | ≤¥60 |
| 服务器成本/用户 | 平均每用户服务器成本 | 服务器总成本 / 总用户数 | 每月 | ≤¥0.5 |
| 客服成本/用户 | 平均每用户客服成本 | 客服总成本 / 总用户数 | 每月 | ≤¥0.2 |

## 四、数据分析方法

### 4.1 用户分群分析（Cohort Analysis）

**目的：** 追踪不同时期用户的行为差异和留存趋势

**分析方法：**
```python
# 按注册周进行分群
cohorts = users.groupby('register_week')

# 计算各分群在第 N 周的留存率
for cohort_name, cohort_data in cohorts:
    week_0_users = len(cohort_data)
    for week_n in range(1, 13):
        week_n_users = count_active_users(cohort_data, week_n)
        retention_rate = week_n_users / week_0_users
        save_to_heatmap(cohort_name, week_n, retention_rate)

# 输出：留存热力图
```

**应用场景：**
- 评估产品改版对留存的影响
- 对比不同渠道用户质量
- 识别最佳留存窗口期

### 4.2 漏斗分析（Funnel Analysis）

**目的：** 识别转化路径中的流失节点

**关键漏斗：**

#### 注册转化漏斗
```
访问首页 → 点击注册 → 完成注册 → 完善资料 → 首次发布
  100%      65%        55%        45%        30%
```

#### 付费转化漏斗
```
曝光付费点 → 点击 → 查看定价 → 试用 → 付费
    100%     15%      10%      6%      4.8%
```

#### 社交匹配漏斗
```
进入匹配 → 完成测评 → 匹配成功 → 发起聊天 → 持续聊天
   100%      80%       50%       20%       12%
```

**分析方法：**
- 计算各步骤转化率
- 识别流失最大的节点
- A/B 测试优化方案
- 追踪转化率趋势

### 4.3 留存分析（Retention Analysis）

**目的：** 理解用户留存规律，找到提升空间

**分析方法：**

#### 1. 同期群留存曲线
- 横轴：用户注册后的天数/周数
- 纵轴：留存率
- 多条曲线：不同时期注册的用户

#### 2. 魔法数字分析（Magic Number）
找出与高留存强相关的用户行为：
```python
# 分析不同行为次数与 30 日留存的关系
for action in ['diary_post', 'like', 'comment', 'share']:
    for threshold in range(1, 20):
        users_with_action = count_users(action_count >= threshold)
        retention = calc_30d_retention(users_with_action)
        if retention > 0.5:  # 找到留存率>50% 的阈值
            print(f"{action} >= {threshold}: retention = {retention}")

# 输出示例：发布日记 >= 5 篇的用户，30 日留存率达 52%
```

#### 3. 留存归因分析
- 分析各功能使用对留存的贡献
- 计算功能渗透率与留存率的相关性
- 识别关键留存驱动因素

### 4.4 RFM 用户价值分析

**目的：** 基于用户价值进行精细化运营

**RFM 维度：**
| 维度 | 定义 | 分档 |
|------|------|------|
| R（Recency） | 最近一次活跃时间 | 1-5 分（越近越高） |
| F（Frequency） | 活跃频率 | 1-5 分（越高越高） |
| M（Monetary） | 消费金额 | 1-5 分（越高越高） |

**用户分层：**
| 用户类型 | RFM 特征 | 运营策略 |
|---------|---------|---------|
| 重要价值用户 | R 高 F 高 M 高 | VIP 服务、专属权益 |
| 重要发展用户 | R 高 F 高 M 低 | 引导付费、提升 ARPU |
| 重要保持用户 | R 低 F 高 M 高 | 召回激活、防止流失 |
| 重要挽留用户 | R 低 F 低 M 高 | 强力召回、优惠激励 |
| 一般用户 | 中等 | 常规运营、培养习惯 |
| 新用户 | R 高 F 低 M 低 | 新手引导、首单激励 |

### 4.5 A/B 测试分析

**目的：** 科学验证产品/运营策略效果

**测试流程：**
```
1. 提出假设 → 2. 设计实验 → 3. 确定样本量 → 4. 分流实验
                                              ↓
8. 决策落地 ← 7. 得出结论 ← 6. 统计检验 ← 5. 收集数据
```

**样本量计算：**
```python
# 使用功效分析计算所需样本量
from statsmodels.stats.power import zt_ind_solve_power

# 参数设置
baseline_rate = 0.05  # 基准付费率 5%
mde = 0.005  # 最小可检测效应 0.5pp
alpha = 0.05  # 显著性水平
power = 0.8  # 统计功效

# 计算样本量
sample_size = zt_ind_solve_power(
    effect_size=calculate_cohen_d(baseline_rate, mde),
    alpha=alpha,
    power=power,
    ratio=1.0  # 对照组和实验组 1:1
)
print(f"每组需要样本量：{sample_size:.0f}")
```

**显著性检验：**
- 使用 Z 检验或 T 检验
- 置信水平 95%（p < 0.05）
- 考虑多重检验校正
