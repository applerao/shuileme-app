# 推送权限申请流程

**版本：** v1.0  
**日期：** 2026-03-07  
**适用项目：** 睡了么 App

---

## 1. 概述

本文档详细说明"睡了么"App 在各平台申请推送通知权限的完整流程，包括 iOS App Store、Android 各大应用商店的推送权限配置和上架审核要求。

## 2. iOS 推送权限申请

### 2.1 Apple Developer 账号准备

#### 2.1.1 账号类型选择

| 账号类型 | 费用 | 适用场景 | 推荐度 |
|----------|------|----------|--------|
| 个人账号 | $99/年 | 个人开发者 | ⭐⭐ |
| 公司账号 | $99/年 | 企业/团队 | ⭐⭐⭐⭐⭐ |
| 企业账号 | $299/年 | 内部分发 | ⭐ |

> ✅ **推荐：** 公司账号（支持多成员协作，可上架 App Store）

#### 2.1.2 账号注册流程

1. 访问 [Apple Developer](https://developer.apple.com/)
2. 点击 "Account" → "Enroll Now"
3. 选择账号类型（推荐 Company/Organization）
4. 填写公司信息（需要邓白氏编码 D-U-N-S Number）
5. 等待审核（通常 3-5 个工作日）
6. 审核通过后支付年费

#### 2.1.3 获取邓白氏编码

1. 访问 [邓白氏编码申请](https://www.dnb.com.cn/duns/)
2. 填写企业信息
3. 等待审核（约 5-7 个工作日）
4. 获取 9 位 D-U-N-S 编码

### 2.2 推送证书申请

#### 2.2.1 创建 Certificate Signing Request (CSR)

1. 打开 Mac 上的"钥匙串访问"
2. 菜单栏 → 证书助理 → 从证书颁发机构请求证书
3. 填写邮箱（与 Apple ID 一致）
4. 选择"存储到磁盘"
5. 保存 `CertificateSigningRequest.certSigningRequest`

#### 2.2.2 创建 App ID

1. 登录 [Certificates, IDs & Profiles](https://developer.apple.com/account/resources/identifiers/list)
2. 点击 "+" → "App IDs" → "App"
3. 配置：
   ```
   Description: SleptApp
   Bundle ID: com.shuileme.slept
   Capabilities: 
     ✓ Push Notifications
     ✓ Associated Domains (如需 Universal Link)
   ```

#### 2.2.3 创建推送证书

**开发证书（Sandbox）：**

1. 进入 "Certificates" → "+" → "Apple Push Notification service SSL (Sandbox)"
2. 选择 App ID: `com.shuileme.slept`
3. 上传 CSR 文件
4. 下载证书 `aps_development.cer`
5. 双击安装到钥匙串

**生产证书（Production）：**

1. 进入 "Certificates" → "+" → "Apple Push Notification service SSL (Production)"
2. 选择 App ID: `com.shuileme.slept`
3. 上传 CSR 文件
4. 下载证书 `aps_production.cer`
5. 双击安装到钥匙串

#### 2.2.4 导出 .p12 证书

1. 打开"钥匙串访问"
2. 筛选"证书"，找到 "Apple Push Services: com.shuileme.slept"
3. 右键 → 导出
4. 选择格式 `.p12`，设置密码
5. 保存：
   - `slept_push_dev.p12` (开发)
   - `slept_push_prod.p12` (生产)

> ⚠️ **重要：** 记住导出密码，后续配置需要用到！

### 2.3 创建 Provisioning Profile

#### 2.3.1 开发 Profile

1. 进入 "Profiles" → "+" → "iOS App Development"
2. 选择 App ID: `com.shuileme.slept`
3. 选择开发证书
4. 选择测试设备（需先注册 UDID）
5. 命名：`SleptApp Dev Profile`
6. 下载并安装到 Xcode

#### 2.3.2 发布 Profile（App Store）

1. 进入 "Profiles" → "+" → "App Store"
2. 选择 App ID: `com.shuileme.slept`
3. 选择发布证书
4. 命名：`SleptApp App Store Profile`
5. 下载并安装到 Xcode

#### 2.3.3 注册测试设备 UDID

1. 连接 iPhone 到 Mac
2. 打开 Xcode → Window → Devices and Simulators
3. 选择设备，复制 UDID
4. 进入 "Devices" → "+" 添加设备
5. 填写设备名称和 UDID

### 2.4 Xcode 项目配置

#### 2.4.1 开启 Push Notifications 能力

1. 打开 Xcode 项目
2. 选择 Target → "Signing & Capabilities"
3. 点击 "+ Capability" → 搜索 "Push Notifications"
4. 自动配置或手动导入 Profile

#### 2.4.2 配置 Background Modes

1. Target → "Signing & Capabilities"
2. "+ Capability" → "Background Modes"
3. 勾选 "Remote notifications"

#### 2.4.3 配置 Info.plist

```xml
<key>UIBackgroundModes</key>
<array>
    <string>remote-notifications</string>
</array>

<key>NSUserNotificationsUsageDescription</key>
<string>我们需要通知权限来提醒您按时睡觉和起床打卡</string>
```

### 2.5 App Store 上架审核

#### 2.5.1 审核要求

Apple 对推送通知的审核要求：

1. **明确说明用途：** 在隐私政策中说明推送用途
2. **用户可关闭：** 必须允许用户在系统设置中关闭通知
3. **不得滥用：** 不能频繁推送营销内容
4. **内容合规：** 推送内容需符合 App 功能

#### 2.5.2 提交审核材料

在 App Store Connect 提交审核时，需在"备注"中说明：

```
推送通知使用说明：

1. 打卡提醒：用户可自定义提醒时间，用于提醒入睡和起床打卡
2. 监督提醒：用户加入睡眠小组后，可接收组员的打卡提醒
3. 系统通知：账号安全、功能更新等重要通知

用户可在 App 内设置中关闭各类通知，也可在 iOS 系统设置中统一管理。
推送频率控制在每日不超过 3 条，避免打扰用户。
```

#### 2.5.3 常见审核被拒原因

| 原因代码 | 原因描述 | 解决方案 |
|----------|----------|----------|
| 2.1 | App 不完整 | 确保所有功能可用，无占位内容 |
| 5.1.1 | 隐私问题 | 完善隐私政策，说明推送用途 |
| 3.1.1 | 隐藏功能 | 推送功能需在 App 内有对应入口 |
| 4.3 | 重复应用 | 确保应用有独特功能 |

## 3. Android 推送权限申请

### 3.1 软件著作权申请（中国大陆上架必备）

#### 3.1.1 准备材料

1. **申请表：** 在中国版权保护中心网站填写
2. **主体资格证明：**
   - 公司：营业执照复印件（加盖公章）
   - 个人：身份证复印件
3. **源代码：** 前后各 30 页，共 60 页（每页 50 行）
4. **用户手册：** 功能说明截图（15 页以上）
5. **身份证明：** 经办人身份证复印件

#### 3.1.2 申请流程

1. 访问 [中国版权保护中心](http://www.ccopyright.com.cn/)
2. 注册账号并实名认证
3. 在线填写申请表
4. 上传材料
5. 提交后打印申请表，盖章后邮寄
6. 等待审核（约 30-60 个工作日）
7. 审核通过后领取证书

#### 3.1.3 加急办理

| 办理时长 | 费用 | 说明 |
|----------|------|------|
| 普通 | 免费 | 30-60 工作日 |
| 加急 30 日 | ¥1,000 | 30 工作日 |
| 加急 20 日 | ¥1,500 | 20 工作日 |
| 加急 10 日 | ¥2,500 | 10 工作日 |

> 💡 **建议：** 提前 2 个月开始申请，选择加急服务

### 3.3 Android 13+ 通知权限适配

从 Android 13 (API 33) 开始，通知权限需要动态申请：

#### 3.3.1 Manifest 配置

```xml
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
```

#### 3.3.2 动态申请代码

```javascript
// React Native
import { PermissionsAndroid, Platform } from 'react-native';

export async function requestNotificationPermission() {
  if (Platform.OS === 'android' && Platform.Version >= 33) {
    try {
      const granted = await PermissionsAndroid.request(
        PermissionsAndroid.PERMISSIONS.POST_NOTIFICATIONS,
        {
          title: '通知权限申请',
          message: '睡了么需要使用通知权限来提醒您按时睡觉和起床打卡',
          buttonPositive: '允许',
          buttonNegative: '拒绝',
        },
      );
      return granted === PermissionsAndroid.RESULTS.GRANTED;
    } catch (err) {
      console.warn(err);
      return false;
    }
  }
  return true;
}
```

#### 3.3.3 权限引导弹窗

在首次启动时展示权限引导：

```javascript
// 权限引导文案
const PERMISSION_GUIDE = {

我们不会发送无关的营销通知`,
```

## 4. 推送通知分类配置

### 4.1 iOS 通知分类 (Notification Categories)

```objective-c
// 在 AppDelegate 中配置通知分类
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    
    UNUserNotificationCenter *center = [UNUserNotificationCenter currentNotificationCenter];
    
    // 打卡提醒分类
    UNMutableNotificationCategory *checkinCategory = [[UNMutableNotificationCategory alloc] initWithIdentifier:@"CHECKIN_CATEGORY" actions:@[] intentIdentifiers:@[] options:UNNotificationCategoryOptionNone];
    
    // 监督提醒分类（带操作按钮）
    UNNotificationAction *remindAction = [UNNotificationAction actionWithIdentifier:@"REMIND_ACTION" title:@"提醒 TA" options:UNNotificationActionOptionForeground];
    UNMutableNotificationCategory *supervisionCategory = [[UNMutableNotificationCategory alloc] initWithIdentifier:@"SUPERVISION_CATEGORY" actions:@[remindAction] intentIdentifiers:@[] options:UNNotificationCategoryOptionNone];
    
    center.delegate = self;
    [center setNotificationCategories:[NSSet setWithObjects:checkinCategory, supervisionCategory, nil]];
    
    return YES;
}
```

### 4.2 Android 通知渠道 (Notification Channels)

```javascript
// React Native - 创建通知渠道
import notifee from '@notifee/react-native';

export async function createNotificationChannels() {
  // 打卡提醒渠道
  await notifee.createChannel({
    id: 'checkin',
    name: '打卡提醒',
    importance: AndroidImportance.HIGH,
    vibration: true,
    sound: 'default',
  });

  // 监督提醒渠道
  await notifee.createChannel({
    id: 'supervision',
    name: '监督提醒',
    importance: AndroidImportance.HIGH,
    vibration: true,
    sound: 'default',
  });

  // 系统通知渠道
  await notifee.createChannel({
    id: 'system',
    name: '系统通知',
    importance: AndroidImportance.DEFAULT,
    vibration: false,
  });

  // 睡眠报告渠道
  await notifee.createChannel({
    id: 'report',
    name: '睡眠报告',
    importance: AndroidImportance.DEFAULT,
    vibration: false,
  });
}
```

### 4.3 通知优先级配置

| 通知类型 | iOS 优先级 | Android 重要性 | 是否播放声音 |
|----------|-----------|---------------|-------------|
| 打卡提醒 | High | High | ✅ |
| 监督提醒 | High | High | ✅ |
| 系统通知 | Default | Default | ❌ |
| 睡眠报告 | Default | Low | ❌ |
| 营销推送 | Low | Low | ❌ |

### 5.1 隐私政策模板（推送相关条款）

```markdown

### 6.4 推送通知

为了向您提供打卡提醒、监督提醒等服务，我们会收集您的设备信息和通知权限。

**收集的信息：**
- 设备标识符（用于推送目标识别）

**使用目的：**
- 发送打卡提醒，帮助您养成规律作息
- 发送监督提醒，实现社交监督功能
- 发送系统通知，如账号安全、功能更新等

**关闭方式：**
您可在 App 内"设置 → 通知设置"中关闭各类通知，也可在手机系统设置中关闭睡了么的通知权限。关闭后可能影响部分功能使用。

**第三方服务：**
我们使用极光推送 (JPush) 提供推送服务，极光推送会收集必要的设备信息以完成推送。详见 [极光推送隐私政策](https://www.jiguang.cn/license/privacy)。
```

## 7. 常见问题

### 7.1 iOS 审核被拒 - 推送权限说明不清

**问题：** 审核人员反馈推送用途不明确

**解决：**
1. 在审核备注中详细说明推送用途
2. 在 App 内添加通知设置页面
3. 确保隐私政策包含推送说明

**回复模板：**
```
尊敬的审核团队：

睡了么 App 使用推送通知仅用于以下场景：

1. 打卡提醒：用户可自定义提醒时间，用于提醒入睡和起床打卡
2. 监督提醒：用户加入睡眠小组后，可接收组员的打卡提醒
3. 系统通知：账号安全、功能更新等重要通知

用户可在 App 内"设置 → 通知设置"中关闭各类通知，也可在 iOS 系统设置中统一管理。

推送频率控制在每日不超过 3 条，不会发送营销内容。

感谢您的审核！
```

### 7.2 Android 上架被拒 - 缺少软著

**问题：** 应用商店要求提供软件著作权证书

**解决：**
1. 如急迫上架，可先申请加急软著
2. 部分商店可先上架后补材料
3. 联系应用商店客服说明情况

### 7.3 推送权限被用户拒绝

**问题：** 用户首次拒绝后，无法再次申请

**解决：**
1. iOS：引导用户去系统设置手动开启
2. Android 13+：可再次触发系统弹窗
3. 在 App 内添加"开启通知"引导页

**引导文案：**
```
通知权限已关闭

您尚未开启通知权限，将无法接收：
• 睡前打卡提醒
• 监督伙伴提醒
• 睡眠报告推送

[去设置开启] [暂不开启]
```
