# 微信登录集成指南

## 📦 快速开始

### 1. 安装依赖

```bash
cd slept-app
npm install react-native-wechat
```

### 2. 配置微信 AppID

编辑 `src/utils/wechat.ts`，替换 AppID：

```typescript
export const WECHAT_APP_ID = '你的微信 AppID';
```

### 3. 平台配置

#### Android 配置

**android/app/src/main/AndroidManifest.xml**:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application>
        <!-- 微信登录回调 Activity -->
        <activity
            android:name=".wxapi.WXEntryActivity"
            android:exported="true"
            android:launchMode="singleTop"
            android:theme="@android:style/Theme.Translucent.NoTitleBar" />
        
        <!-- 添加微信 SDK 依赖 -->
        <meta-data
            android:name="wechat_app_id"
            android:value="wxXXXXXXXXXXXXXXXX" />
    </application>
</manifest>
```

**android/app/build.gradle**:

```gradle
dependencies {
    implementation 'com.tencent.mm.opensdk:wechat-sdk-android:6.8.0'
}
```

#### iOS 配置

**ios/YourApp/Info.plist**:

```xml
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>weixin</string>
    <string>weixinULAPI</string>
</array>

<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>wxXXXXXXXXXXXXXXXX</string>
        </array>
    </dict>
</array>
```

**ios/YourApp/AppDelegate.mm**:

```objc
#import "RCTWeChat.h"

- (BOOL)application:(UIApplication *)application
            openURL:(NSURL *)url
            options:(NSDictionary<UIApplicationOpenURLOptionsKey,id> *)options
{
    if ([RCTWeChat handleOpenURL:url]) {
        return YES;
    }
    return [super application:application openURL:url options:options];
}

- (BOOL)application:(UIApplication *)application
continueUserActivity:(NSUserActivity *)userActivity
 restorationHandler:(void(^)(NSArray<id<UIUserActivityRestoring>> * __nullable restorableObjects))restorationHandler
{
    if ([RCTWeChat handleOpenURL:userActivity.webpageURL]) {
        return YES;
    }
    return [super application:application continueUserActivity:userActivity restorationHandler:restorationHandler];
}
```

### 4. 初始化微信 SDK

在应用入口文件（如 `App.tsx`）中初始化：

```typescript
import React, { useEffect } from 'react';
import { initWeChat } from './src/utils/wechat';

export default function App() {
  useEffect(() => {
    const init = async () => {
      const success = await initWeChat();
      if (success) {
        console.log('微信 SDK 初始化成功');
      }
    };
    init();
  }, []);

  return (
    // ... your app
  );
}
```

### 5. 使用微信登录按钮

在登录页面使用封装好的组件：

```typescript
import React from 'react';
import { WechatLoginButton } from './src';

function LoginPage() {
  const handleWechatLogin = async () => {
    try {
      // 这里会自动完成微信授权和后端接口调用
      console.log('微信登录成功');
      
      // 可以在这里处理登录成功后的逻辑
      // 如：保存 token、跳转首页等
    } catch (error) {
      console.error('微信登录失败:', error);
    }
  };

  return (
    <WechatLoginButton
      onPress={handleWechatLogin}
      size="medium"
    />
  );
}
```

## 🔧 工具函数

### initWeChat()

```typescript
const success = await initWeChat();
```

### Android

- 需要配置签名指纹
- 需要创建 WXEntryActivity
- 使用 keystore 签名

### iOS

- 需要配置 URL Scheme
- 需要配置 LSApplicationQueriesSchemes
- 需要在 AppDelegate 处理回调

---

## 🔐 安全建议

1. **AppSecret 保护**: 永远不要将 AppSecret 放在客户端代码中
2. **HTTPS**: 生产环境必须使用 HTTPS
3. **Token 存储**: 使用安全的存储方式（如 Keychain/EncryptedStorage）
4. **签名验证**: 确保应用签名与微信开放平台一致
5. **频率限制**: 后端已配置节流，防止暴力攻击

## 🐛 常见问题

### 1. 签名错误

**现象**: `errcode: 40013, errmsg: invalid appid`

**解决**:
- 检查 AppID 是否正确
- 重新生成应用签名
- 确保使用正确的 keystore

### 2. 回调失败

**现象**: 授权后无法返回应用

**解决**:
- Android: 检查 AndroidManifest.xml 配置
- iOS: 检查 URL Scheme 和 AppDelegate
- 确保包名/Bundle ID 与微信开放平台一致

### 3. 网络错误

**现象**: `Failed to exchange WeChat code`

**解决**:
- 检查服务器网络连通性
- 验证 AppSecret 配置
- 检查微信 API 调用频率限制

### 4. 组件不显示图标

**现象**: 微信登录按钮显示异常

**解决**:
- 检查字体/图标库是否正确加载
- 尝试使用图片代替 emoji
- 清除缓存重新编译
