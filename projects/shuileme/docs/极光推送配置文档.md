## 1. 概述

本文档详细说明"睡了么"App 使用极光推送 (JPush) 服务的完整配置流程，包括账号注册、应用创建、证书配置、SDK 集成和推送测试。

## 2. 账号注册与应用创建

### 2.1 注册极光开发者账号

1. 访问 [极光官网](https://www.jiguang.cn/)
2. 点击右上角"注册"，使用手机号注册
3. 完成实名认证（需要营业执照或个人身份证）
4. 登录控制台：https://console.jiguang.cn/

### 2.2 创建应用

1. 登录后进入"控制台" → "推送服务" → "创建应用"
2. 填写应用信息：
   - **应用名称：** 睡了么
   - **应用包名：** 
     - iOS: `com.shuileme.slept`
     - Android: `com.shuileme.slept`
   - **图标：** 上传 App 图标 (512x512)
3. 创建成功后，记录以下关键信息：
   ```
   AppKey: xxxxxxxxxxxxxxxx
   Master Secret: xxxxxxxxxxxxxxxx
   ```

> ⚠️ **安全提示：** Master Secret 仅在后端服务器使用，严禁泄露或写入前端代码！

## 3. iOS 推送配置

### 3.1 申请推送证书

#### 3.1.1 创建 App ID

1. 登录 [Apple Developer](https://developer.apple.com/)
2. 进入 "Certificates, IDs & Profiles"
3. 点击 "Identifiers" → "+" 创建新 App ID
4. 配置：
   - **Description:** SleptApp
   - **Bundle ID:** `com.shuileme.slept`
   - **Capabilities:** 勾选 "Push Notifications"

#### 3.1.2 创建推送证书

1. 在 "Certificates" 页面点击 "+"
2. 选择证书类型：
   - **开发环境:** Apple Push Notification service SSL (Sandbox)
   - **生产环境:** Apple Push Notification service SSL (Production)
3. 上传 CSR 文件（通过钥匙串访问生成）
4. 下载证书 `.cer` 文件，双击安装到钥匙串

#### 3.1.3 导出 .p12 证书

1. 打开"钥匙串访问"
2. 找到刚安装的证书（如 "Apple Push Services: com.shuileme.slept"）
3. 右键 → "导出"
4. 选择格式 `.p12`，设置密码
5. 保存证书文件：
   - `slept_push_dev.p12` (开发)
   - `slept_push_prod.p12` (生产)

### 3.2 配置 Provisioning Profile

1. 进入 "Profiles" → "+" 创建描述文件
2. 选择类型：
   - 开发：iOS App Development
   - 发布：App Store / Ad Hoc
3. 选择 App ID、证书、设备
4. 下载并安装到 Xcode

### 3.3 在极光控制台上传证书

1. 进入极光控制台 → 应用 → 推送设置 → iOS
2. 上传 .p12 证书文件
3. 输入导出时设置的密码
4. 分别上传开发证书和生产证书
5. 保存配置

> ✅ **验证：** 证书状态显示"有效"即配置成功

## 4. Android 推送配置

### 4.1 基础配置

1. 进入极光控制台 → 应用 → 推送设置 → Android
2. 配置包名：`com.shuileme.slept`
3. 上传应用签名（SHA1/SHA256）：
   ```bash
   # 获取签名指纹
   keytool -list -v -keystore slept-release.keystore -alias slept
   ```

为提高推送到达率，建议配置厂商通道：

#### 4.2.1 华为 HMS

2. 创建应用，获取 App ID
3. 下载 `agconnect-services.json`
4. 在极光控制台填写华为 App ID

#### 4.2.2 小米 MiPush

1. 登录 [小米开放平台](https://dev.mi.com/)
2. 创建应用，获取 App ID 和 App Key
3. 在极光控制台填写小米配置

#### 4.2.3 OPPO / VIVO

1. 分别登录对应开发者平台
2. 创建应用获取配置
3. 在极光控制台填写

### 5.2 iOS 配置

```bash
cd ios
pod install
```

#### 5.2.2 配置 Info.plist

在 `Info.plist` 中添加：

```xml
<key>UIBackgroundModes</key>
<array>
    <string>remote-notifications</string>
</array>
```

#### 5.2.3 配置 AppDelegate

```objective-c
// AppDelegate.m
#import "JPush.h"

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    
    [JPUSHService setupWithOption:launchOptions
                           appKey:@"你的 AppKey"
                          channel:@"App Store"
                         apsForProduction:YES];
    
    [JPUSHService registerForRemoteNotificationConfig:launchOptions];
    
    return YES;
}

// 处理通知
- (void)application:(UIApplication *)application didRegisterForRemoteNotificationsWithDeviceToken:(NSData *)deviceToken {
    [JPUSHService registerDeviceToken:deviceToken];
}

- (void)application:(UIApplication *)application didReceiveRemoteNotification:(NSDictionary *)userInfo fetchCompletionHandler:(void (^)(UIBackgroundFetchResult))completionHandler {
    [JPUSHService handleRemoteNotification:userInfo];
    completionHandler(UIBackgroundFetchResultNewData);
}
```

### 5.3 Android 配置

#### 5.3.1 AndroidManifest.xml

```xml
<manifest>
    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.WAKE_LOCK" />
    <uses-permission android:name="android.permission.VIBRATE" />
    <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
    
    <application>
        <!-- JPush 配置 -->
        <meta-data android:name="JPUSH_CHANNEL" android:value="${JPUSH_CHANNEL}" />
        <meta-data android:name="JPUSH_APPKEY" android:value="${JPUSH_APPKEY}" />
        
        <!-- JPush Service -->
        <service
            android:name="cn.jpush.android.service.PushService"
            android:enabled="true"
            android:exported="false" />
            
        <!-- JPush Receiver -->
        <receiver
            android:name="cn.jpush.android.service.PushReceiver"
            android:enabled="true"
            android:exported="false">
            <intent-filter>
                <action android:name="cn.jpush.android.intent.RECEIVE_MESSAGE" />
            </intent-filter>
        </receiver>
    </application>
</manifest>
```

### 5.4 React Native 调用示例

```javascript
// src/services/PushService.js
import JPush from 'jpush-react-native';

export class PushService {
  // 初始化推送
  static init() {
    JPush.init({
      appKey: '你的 AppKey',
      channel: 'App Store', // 或应用市场名称
      production: __DEV__ ? false : true,
    });

    // 获取 Registration ID
    JPush.getRegistrationID((result) => {
      console.log('Registration ID:', result);
      // 将 registrationId 发送到后端绑定用户
      this.bindDevice(result);
    });

    // 监听通知点击
    JPush.addNotificationListener((result) => {
      console.log('通知点击:', result);
      // 根据 notificationId 跳转对应页面
      this.handleNotificationClick(result);
    });
  }

  // 绑定设备到用户
  static async bindDevice(registrationId) {
    const token = await AsyncStorage.getItem('token');
    fetch('https://api.shuile.me/v1/device/register', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        deviceId: registrationId,
        deviceType: Platform.OS === 'ios' ? 1 : 2,
        pushToken: registrationId,
        appVersion: '1.0.0',
      }),
    });
  }

  // 设置别名（用户 ID）
  static setAlias(userId) {
    JPush.setAlias({
      alias: userId.toString(),
      callback: (result) => {
        console.log('别名设置:', result);
      },
    });
  }

  // 添加标签
  static addTags(tags) {
    JPush.addTags({
      tags: tags, // 如 ['insomnia', 'early_riser']
      callback: (result) => {
        console.log('标签添加:', result);
      },
    });
  }
}
```

### 6.3 创建推送服务

```typescript
// src/modules/notification/jpush.service.ts
import JPush from 'jpush-sdk';
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class JpushService {
  private client: JPush;

  constructor(private configService: ConfigService) {
    this.client = new JPush(
      this.configService.get('JPUSH_APP_KEY'),
      this.configService.get('JPUSH_MASTER_SECRET'),
    );
  }

  // 推送给单个用户（通过别名）
  async pushToUser(userId: string, title: string, content: string, extras?: any) {
    const payload = this.client.payload();
    
    payload.setPlatform('all');
    payload.setAudience(JPush.alias(userId));
    
    payload.addNotification(
      JPush.notification(title, content, '睡了么', extras),
    );

    try {
      const result = await this.client.send(payload);
      console.log('推送成功:', result);
      return { success: true, result };
    } catch (error) {
      console.error('推送失败:', error);
      return { success: false, error };
    }
  }

  // 推送给多个用户
  async pushToUsers(userIds: string[], title: string, content: string, extras?: any) {
    const payload = this.client.payload();
    
    payload.setPlatform('all');
    payload.setAudience(JPush.alias(userIds));
    
    payload.addNotification(
      JPush.notification(title, content, '睡了么', extras),
    );

    try {
      const result = await this.client.send(payload);
      return { success: true, result };
    } catch (error) {
      return { success: false, error };
    }
  }

  // 推送给所有用户（广播）
  async pushToAll(title: string, content: string, extras?: any) {
    const payload = this.client.payload();
    
    payload.setPlatform('all');
    payload.setAudience(JPush.all());
    
    payload.addNotification(
      JPush.notification(title, content, '睡了么', extras),
    );

    try {
      const result = await this.client.send(payload);
      return { success: true, result };
    } catch (error) {
      return { success: false, error };
    }
  }

  // 推送给指定标签用户
  async pushToTag(tag: string, title: string, content: string, extras?: any) {
    const payload = this.client.payload();
    
    payload.setPlatform('all');
    payload.setAudience(JPush.tag(tag));
    
    payload.addNotification(
      JPush.notification(title, content, '睡了么', extras),
    );

    try {
      const result = await this.client.send(payload);
      return { success: true, result };
    } catch (error) {
      return { success: false, error };
    }
  }
}
```

## 7. 推送权限申请

### 7.1 iOS 权限申请

在 `AppDelegate.m` 中配置：

```objective-c
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    
    // 请求通知权限
    UNUserNotificationCenter *center = [UNUserNotificationCenter currentNotificationCenter];
    UNAuthorizationOptions options = UNAuthorizationOptionAlert | 
                                     UNAuthorizationOptionSound | 
                                     UNAuthorizationOptionBadge;
    
    [center requestAuthorizationWithOptions:options
                          completionHandler:^(BOOL granted, NSError * _Nullable error) {
        if (granted) {
            NSLog(@"通知权限已授予");
        } else {
            NSLog(@"通知权限被拒绝: %@", error);
        }
    }];
    
    return YES;
}
```

### 7.2 Android 权限申请

在 `AndroidManifest.xml` 中声明：

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.WAKE_LOCK" />
<uses-permission android:name="android.permission.VIBRATE" />
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" /> <!-- Android 13+ -->
```

在代码中动态申请（Android 13+）：

```javascript
// React Native
import { PermissionsAndroid, Platform } from 'react-native';

async function requestNotificationPermission() {
  if (Platform.OS === 'android' && Platform.Version >= 33) {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.POST_NOTIFICATIONS,
    );
    return granted === PermissionsAndroid.RESULTS.GRANTED;
  }
  return true;
}
```

## 9. 常见问题排查

### 9.1 iOS 推送失败

**问题：** 开发环境推送成功，生产环境失败

**排查：**
1. 检查是否上传了生产证书
2. 检查证书是否过期（有效期 1 年）
3. 检查 App 是否使用生产证书打包
4. 检查 `apns_production` 参数设置

**解决：**
```javascript
// 生产环境配置
JPush.init({
  production: true,  // 必须为 true
});

// 后端推送配置
"options": {
  "apns_production": true
}
```

### 9.2 Android 推送到达率低

**问题：** 部分用户收不到推送

**原因：**
- 应用被杀死后，普通推送无法到达
- 部分厂商限制后台进程

**解决：**
1. 配置厂商通道（华为/小米/OPPO/VIVO）
2. 引导用户将 App 加入白名单
3. 使用高优先级推送

### 9.3 别名设置失败

**问题：** `setAlias` 调用失败

**排查：**
1. 检查别名格式（建议纯数字或字母）
2. 检查网络是否通畅
3. 检查 SDK 版本是否最新

**解决：**
```javascript
// 别名格式规范
JPush.setAlias({
  alias: 'user_12345',  // 避免特殊字符
  callback: (result) => {
    if (result.code !== 0) {
      console.error('别名设置失败:', result);
      // 重试逻辑
      setTimeout(() => this.setAlias(userId), 3000);
    }
  },
});
```

### 9.4 推送重复接收

**问题：** 同一条推送收到多次

**原因：**
- 多个设备绑定同一用户
- 推送逻辑重复调用

**解决：**
1. 后端记录已推送的消息 ID，避免重复
2. 前端根据 messageId 去重
3. 清理无效设备绑定

## 10. 推送数据统计

### 10.1 关键指标

在极光控制台可查看：

- **推送到达率：** 成功送达设备数 / 发送总数
- **推送打开率：** 点击通知数 / 送达数
- **设备在线率：** 实时在线设备比例

### 10.2 数据埋点

```javascript
// 推送点击埋点
```

### 10.3 优化建议

1. **推送时间：** 根据用户活跃时段发送（晚 20:00-22:00 最佳）
2. **推送文案：** 个性化文案打开率提升 30%
3. **推送频率：** 单用户每日不超过 3 条，避免打扰
4. **A/B 测试：** 测试不同文案的打开率
