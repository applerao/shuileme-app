## 值班职责

### 日常职责

1. **告警响应**
   - P0 告警：5 分钟内响应
   - P1 告警：30 分钟内响应
   - P2 告警：2 小时内响应

2. **监控巡检**
   - 每日 3 次查看 Dashboard (10:00/14:00/18:00)
   - 检查告警趋势
   - 确认无异常指标

3. **问题处理**
   - 初步诊断问题
   - 执行标准处理流程
   - 必要时升级

4. **交接记录**
   - 记录当班期间事件
   - 更新待跟进问题
   - 周一交接会议

### 交接班流程

```
交班前 (周日 18:00 前)
    ↓
填写交接文档
    ↓
标记待跟进问题
    ↓
周一交接会议 (10:00)
    ↓
接班人员确认
    ↓
完成交接
```

## 响应流程

### P0 告警响应

```
1. 收到告警 (电话 + 短信 + 钉钉)
   ↓
2. 5 分钟内确认告警
   ↓
3. 初步诊断 (5-10 分钟)
   ↓
4. 执行应急处理
   - 服务重启
   - 流量切换
   - 回滚发布
   ↓
5. 15 分钟内同步进展
   ↓
6. 问题解决后写复盘报告
```

### P1 告警响应

```
1. 收到告警 (短信 + 钉钉)
   ↓
2. 30 分钟内确认告警
   ↓
3. 诊断问题根因
   ↓
4. 制定处理方案
   ↓
5. 执行修复
   ↓
6. 验证恢复
```

### P2 告警响应

```
1. 收到告警 (钉钉)
   ↓
2. 2 小时内确认
   ↓
3. 评估影响范围
   ↓
4. 安排处理时间
   ↓
5. 工作时间处理
```

## 复盘要求

### P0 告警

- **时间**: 24 小时内
- **参与**: 全体相关人员
- **输出**: 事故报告 + 改进措施

### P1 告警

- **时间**: 3 天内
- **参与**: 值班人员 + 相关开发
- **输出**: 问题分析 + 优化建议

### P2 告警

- **时间**: 周会同步
- **参与**: 团队内部
- **输出**: 记录到知识库
