# 告警规则详解

## 告警级别定义

| 级别 | 名称 | 响应时间 | 通知渠道 | 升级策略 |
|------|------|----------|----------|----------|
| P0 | Critical | 5 分钟 | 电话 + 短信 + 钉钉 | 15 分钟未响应升级至主管 |
| P1 | High | 30 分钟 | 短信 + 钉钉 | 1 小时未响应升级至 P0 |
| P2 | Medium | 2 小时 | 钉钉 | 4 小时未响应升级至 P1 |

---

## 应用性能告警

### API 响应时间

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| APIResponseTimeP95High | P1 | P95 > 500ms | API 响应时间偏高，影响用户体验 |
| APIResponseTimeP99Critical | P0 | P99 > 1s | API 响应时间严重偏高，需立即处理 |

**排查步骤:**
1. 检查 Grafana API 性能 Dashboard
2. 查看慢请求日志 (ES 中搜索 `performance_tier: slow`)
3. 检查数据库慢查询
4. 检查外部依赖服务状态

### API 错误率

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| APIErrorRateHigh | P1 | > 1% | API 5xx 错误率偏高 |
| APIErrorRateCritical | P0 | > 5% | API 错误率严重，可能服务故障 |

**排查步骤:**
1. 查看错误日志 (ES 中搜索 `severity: high`)
2. 检查应用异常堆栈
3. 检查数据库连接状态
4. 检查外部 API 依赖

### API 可用性

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| APIEndpointDown | P0 | 探测失败 | API 端点完全不可用 |
| APIEndpointSlow | P2 | > 3s | API 端点响应缓慢 |

## 系统资源告警

### CPU 使用率

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| HighCPUUsage | P2 | > 80% | CPU 使用率偏高 |
| CriticalCPUUsage | P1 | > 95% | CPU 使用率严重偏高 |

**排查步骤:**
1. `top` 查看进程 CPU 占用
2. `pidstat` 分析具体线程
3. 检查是否有异常进程
4. 考虑扩容或优化代码

### 内存使用率

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| HighMemoryUsage | P2 | > 85% | 内存使用率偏高 |
| CriticalMemoryUsage | P1 | > 95% | 内存使用率严重偏高 |
| OOMKillingRisk | P0 | 可用 < 5% | 存在 OOM 风险 |

**排查步骤:**
1. `free -m` 查看内存状态
2. `ps aux --sort=-%mem` 查看进程内存
3. 检查是否有内存泄漏
4. 考虑增加内存或优化

### 磁盘使用率

| 告警名称 | 级别 | 阈值 | 说明 |
|----------|------|------|------|
| HighDiskUsage | P2 | > 80% | 磁盘使用率偏高 |
| CriticalDiskUsage | P1 | > 90% | 磁盘使用率严重偏高 |
| DiskWillFillIn24h | P2 | 预测 24h 满 | 磁盘将在 24 小时内写满 |

**排查步骤:**
1. `df -h` 查看磁盘使用
2. `du -sh /*` 查找大目录
3. 清理日志文件
4. 扩容磁盘

## 告警抑制规则

1. **P0 抑制 P1/P2**: 如果存在 P0 告警，同实例的 P1/P2 告警将被抑制
2. **宕机抑制**: 如果 API 端点宕机，该端点的性能告警将被抑制
3. **维护窗口**: 可在维护期间临时静音告警

## 告警升级流程

```
告警触发
    ↓
通知值班人员
    ↓
15 分钟未响应 → 升级至备份值班
    ↓
30 分钟未响应 → 升级至主管
    ↓
1 小时未响应 → 升级至技术负责人
```
