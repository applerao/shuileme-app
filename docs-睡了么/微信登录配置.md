# 微信登录配置文档

## 1. 微信公众平台配置

### 1.1 创建移动应用

1. 访问 [微信开放平台](https://open.weixin.qq.com/)
2. 登录开发者账号
3. 进入「管理中心」→「移动应用」→「创建移动应用」
4. 填写应用信息：
   - 应用名称：睡了么
   - 应用图标：上传 256x256 像素图标
   - 应用类型：社交 - 睡眠健康
   - 备注：睡眠追踪和健康管理应用

### 1.2 获取 AppID 和 AppSecret

创建成功后，在应用详情页面获取：
- **AppID**: `wxXXXXXXXXXXXXXXXX`
- **AppSecret**: `XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`

### 1.3 配置应用签名

#### Android 签名配置

1. 生成签名指纹：
```bash
# 使用 keystore 生成签名
keytool -list -v -keystore ~/android/app/shuileme.keystore -alias shuileme
```

2. 在微信开放平台填写：
   - 应用包名：`com.shuileme`
   - 应用签名：使用「签名生成工具」获取 MD5 值（去掉冒号，小写）

#### iOS 签名配置

1. Bundle ID: `com.shuileme.app`
2. 确保使用正确的 Provisioning Profile

### 1.4 配置回调地址

微信登录回调地址（用于网页授权）：
```
https://api.shuileme.com/api/v1/auth/wechat/callback
```

## 3. 前端配置

### 3.1 安装微信 SDK

```bash
cd shuileme-app
npm install react-native-wechat
```

### 3.2 Android 配置

#### 3.2.1 AndroidManifest.xml

在 `android/app/src/main/AndroidManifest.xml` 添加：

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application>
        <!-- 微信登录回调 Activity -->
        <activity
            android:name=".wxapi.WXEntryActivity"
            android:exported="true"
            android:launchMode="singleTop"
            android:theme="@android:style/Theme.Translucent.NoTitleBar" />
    </application>
</manifest>
```

#### 3.2.2 build.gradle

在 `android/app/build.gradle` 添加依赖：

```gradle
dependencies {
    implementation 'com.tencent.mm.opensdk:wechat-sdk-android:+'
}
```

### 3.3 iOS 配置

#### 3.3.1 Info.plist

在 `ios/shuileme/Info.plist` 添加：

```xml
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>weixin</string>
    <string>weixinULAPI</string>
</array>

<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>wxXXXXXXXXXXXXXXXX</string>
        </array>
    </dict>
</array>
```

#### 3.3.2 AppDelegate

在 `ios/shuileme/AppDelegate.mm` 添加：

```objc
#import "RCTWeChat.h"

- (BOOL)application:(UIApplication *)application
            openURL:(NSURL *)url
            options:(NSDictionary<UIApplicationOpenURLOptionsKey,id> *)options
{
    if ([RCTWeChat handleOpenURL:url]) {
        return YES;
    }
    return [super application:application openURL:url options:options];
}

- (BOOL)application:(UIApplication *)application
continueUserActivity:(NSUserActivity *)userActivity
 restorationHandler:(void(^)(NSArray<id<UIUserActivityRestoring>> * __nullable restorableObjects))restorationHandler
{
    if ([RCTWeChat handleOpenURL:userActivity.webpageURL]) {
        return YES;
    }
    return [super application:application continueUserActivity:userActivity restorationHandler:restorationHandler];
}
```

### 3.4 初始化微信 SDK

在应用启动时（如 App.tsx 或 main entry）调用初始化：

```typescript
import { initWeChat } from './src/utils/wechat';

// 在应用启动时初始化
useEffect(() => {
  const init = async () => {
    await initWeChat();
  };
  init();
}, []);
```

## 4. 测试流程

### 4.1 开发环境测试

1. 确保微信 App 已安装在手机/模拟器
2. 启动后端服务
3. 启动前端应用
4. 点击「微信登录」按钮
5. 跳转到微信授权页面
6. 确认授权
7. 返回应用并获取用户信息

### 4.2 测试用例

#### 测试用例 1: 正常登录流程
```
前置条件：已安装微信 App，微信账号已登录
操作步骤：
1. 打开应用，进入登录页面
2. 点击「微信登录」按钮
3. 跳转到微信授权页面
4. 点击「同意」授权
5. 返回应用

预期结果：
- 成功跳转到微信
- 授权后返回应用
- 显示登录成功提示
- 获取到用户信息和 token
```

#### 测试用例 2: 未安装微信
```
前置条件：设备未安装微信 App
操作步骤：
1. 打开应用，进入登录页面
2. 点击「微信登录」按钮

预期结果：
- 按钮显示「未安装微信」
- 弹出提示引导安装微信
- 不执行登录流程
```

#### 测试用例 3: 用户取消授权
```
前置条件：已安装微信 App
操作步骤：
1. 打开应用，进入登录页面
2. 点击「微信登录」按钮
3. 在微信授权页面点击「取消」

预期结果：
- 返回应用
- 不显示错误提示（静默处理）
- 可以重新尝试登录
```

#### 测试用例 4: 网络错误
```
前置条件：网络断开或后端服务不可用
操作步骤：
1. 断开网络连接
2. 点击「微信登录」按钮
3. 完成微信授权

预期结果：
- 微信授权成功
- 调用后端接口失败
- 显示友好的错误提示
```

### 4.2 常见问题排查

#### 问题 1: 签名错误

**现象**: `errcode: 40013, errmsg: invalid appid`

**解决**:
- 检查 AppID 是否正确
- 重新生成应用签名
- 确保使用正确的 keystore

#### 问题 2: 回调失败

**现象**: 授权后无法返回应用

**解决**:
- 检查 AndroidManifest.xml 配置
- 检查 iOS URL Scheme 配置
- 确保包名/Bundle ID 与微信开放平台一致

#### 问题 3: 网络错误

**现象**: `Failed to exchange WeChat code`

**解决**:
- 检查服务器网络连通性
- 验证 AppSecret 配置
- 检查微信 API 调用频率限制

#### 问题 4: 组件不显示微信图标

**现象**: 微信登录按钮显示异常

**解决**:
- 检查字体/图标库是否正确加载
- 尝试使用图片代替 emoji
- 清除缓存重新编译

#### 问题 5: iOS 回调不生效

**现象**: iOS 授权后无法返回应用

**解决**:
- 检查 Info.plist 中 URL Scheme 配置
- 检查 AppDelegate 中 handleOpenURL 实现
- 确保 URL Scheme 与微信开放平台一致

## 5. 安全建议

1. **AppSecret 保护**: 永远不要将 AppSecret 暴露在客户端代码中
2. **HTTPS**: 生产环境必须使用 HTTPS
3. **Token 验证**: 服务端验证微信返回的 code
4. **频率限制**: 已配置节流，防止暴力攻击
5. **日志记录**: 记录所有登录尝试，便于审计

## 6. API 接口文档

### POST /api/v1/auth/wechat

微信登录接口

**请求参数**:
```json
{
  "wechatCode": "string" // 微信授权码
}
```

**响应示例**:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "userInfo": {
    "id": "uuid",
    "phone": "wechat_xxxxx",
    "nickname": "微信用户",
    "avatar": "https://...",
    "wechatId": "openid_xxxxx",
    "createdAt": "2024-01-01T00:00:00.000Z"
  }
}
```

**错误码**:
- `400`: 微信授权码无效
- `401`: 账户已被禁用
